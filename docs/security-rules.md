# 安全规则

## 1. 禁止操作

- 不允许执行 `rm -rf` 破坏性命令
- 不允许执行 `DROP TABLE`、`DROP DATABASE`
- 不允许执行 `git push --force` 到 main/master
- 不允许修改 Git 历史（`git rebase -i`、`git filter-branch`）
- 不允许清空数据库

## 2. 敏感信息

- 不允许泄露 `.env` 文件内容
- 不允许在代码、注释、日志中硬编码数据库密码
- 不允许在代码、注释、日志中硬编码 JWT Secret
- 不允许在代码、注释、日志中硬编码 API Key、Token
- 不允许提交 `.env` 文件到 Git
- 不允许将密钥发送到外部 API

## 3. 依赖管理

- 不允许随意新增外部依赖
- 新增依赖必须说明：用途、影响范围、是否必要
- 新增依赖前检查是否有现有依赖可替代
- 不允许安装带有已知严重漏洞的依赖版本

## 4. 文件与代码

- 不允许提交 `node_modules/`
- 不允许提交 `dist/`（如有）
- 不允许提交 `.DS_Store`
- 不允许删除用户现有代码和文件
- 不允许删除用户数据库数据

## 5. 代码审查

- 所有 `eval()` 使用必须明确说明原因
- 所有 `innerHTML` 使用必须转义用户输入
- SQL 查询必须使用参数化（Prisma 已自动处理）
- 文件上传必须校验类型和大小

## 6. 网络

- 不允许向未知外部服务器发送用户数据
- 不允许在 C 端 App 中嵌入第三方跟踪脚本
- 管理后台登录必须使用 JWT 鉴权
