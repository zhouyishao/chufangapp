# 产品路线图

## 当前阶段：MVP 联调

- 补齐后台内容管理核心页面。
- 补齐统一后端接口和数据库模型。
- 保证后台配置能在 C 端真实展示。
- 修复核心交互闭环：收藏、浏览、菜篮子、采购、家庭。

## P0：必须稳定

- 登录与鉴权。
- 首页运营配置到 C 端展示。
- 菜谱管理、食材管理、分类标签。
- C 端首页、菜谱详情、食材详情、菜篮子、我的。
- 文件上传与图片展示。

## P1：体验完善

- 搜索筛选。
- 价格记录与趋势展示。
- 家庭共享与成员管理。
- 我的菜谱与上传菜谱流程。
- 采购记录归档。

## P2：后续扩展

- 评论、晒菜、审核。
- 数据报表。
- 资源接入中心。
- 更复杂的运营位和推荐策略。

## 维护规则

- 新功能进入路线图后，先更新本文档，再拆到后台、后端、C 端对应文档。
- 不把详细字段、接口返回、页面 UI 全部堆回 PRD。
