# 后台管理 PRD 范围

本文件承接旧 `docs/admin-cms-prd-dev-spec.md` 的产品范围，避免继续读取 1000 行以上大文档。

## 产品定位

后台管理系统用于维护 C 端展示内容、运营位、分类标签、用户家庭数据和系统资源。

## 核心目标

- 后台配置可真实落库。
- C 端通过接口读取后台配置。
- 内容管理、分类标签、资源管理、用户家庭形成闭环。
- 所有列表具备搜索、筛选、分页、增删改查。

## 信息架构

后台菜单结构见：`docs/admin/menu-structure.md`。

核心模块：

- 工作台。
- 首页运营。
- 内容管理。
- 分类标签。
- 用户与家庭。
- 资源管理。
- 系统设置。

## 开发优先级

### P0

- 登录和鉴权。
- 菜谱、食材、分类标签。
- 首页运营基础内容。
- 后台到 C 端展示联调。

### P1

- Banner、推荐位、时令食材、场景菜单。
- 用户、家庭、采购记录。
- 文件上传和资源管理。

### P2

- 晒菜内容、评论、审核中心。
- 数据报表。
- AI 配置。
