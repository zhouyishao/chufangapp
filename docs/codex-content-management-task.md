## 1. 本次任务目标

继续完成上一个任务，在现有项目基础上，按照 8 张原型图完善后台页面，并跑通：

```text
后台新增 / 编辑内容
→ 保存到后端数据库
→ 后台列表展示
→ C 端接口读取
→ C 端页面展示
→ 后台状态变化影响 C 端展示
```

本次不是重新做项目，不是重构后台，而是在现有项目基础上继续修改和完善。

---

## 2. 重要限制

1. 不要删除已有页面。
2. 不要重构整个项目。
3. 不要重写侧边栏、顶部栏、公共布局。
4. 不要把现有功能推翻重做。
5. 不要随意删掉字段。
6. 不要把新增 / 编辑页改成弹窗。
7. 新增页和编辑页都使用独立页面。
8. 保持当前后台奶油色、绿色、圆角卡片风格。
9. 如果字段后端暂时没有，前端先保留表单字段，后续再接接口，不要删。
10. 如果需要新增组件，优先复用现有公共组件。
11. 不要反复询问普通实现细节，按原型图和本文档继续推进。
12. 遇到非破坏性实现问题，自行判断并继续完成。
13. 遇到删除文件、清空数据、重置数据库等高风险操作时，不要直接执行，改为保留旧文件或做兼容处理。
14. 开始前先执行 `git status`，确认当前工作区状态。
15. 完成后必须运行构建命令，例如 `npm run build`。
16. 如果项目包含多个子项目，例如 `admin-frontend`、`app-frontend`、`server`，请分别运行对应构建和测试命令。

## 4. 统一页面结构要求

所有新增 / 编辑页统一采用以下结构：

1. 左侧后台侧边栏。
2. 顶部面包屑。
3. 顶部搜索框。
4. 右上角按钮：取消、保存草稿、保存并发布。
5. 主内容区使用分块卡片表单。
6. 右侧展示 C 端内容预览。
7. 表单内容修改后，右侧预览应尽量实时更新。
8. 保存后返回对应列表页。
9. 编辑页需要回显已有数据。
10. 列表页点击“新增”进入新增页。
11. 列表页点击“编辑”进入编辑页。
12. 列表页点击“查看”进入详情或预览页。
13. 列表页点击“删除”需要二次确认。
14. 列表页点击“启用 / 禁用 / 上架 / 下架”需要真实改变状态。
15. 列表页搜索、筛选、分页需要能正常使用。

## 5. 内容管理页面详细要求

### 5.1 菜谱管理

参考：`docs/prototypes/05-新增菜谱页.png`

新增菜谱页 / 编辑菜谱页需要包含：

1. 基础信息：菜谱名称、菜谱副标题、菜谱分类、难度、烹饪时长、热量、份量 / 几人食、发布状态、是否推荐、菜谱简介 / 描述。
2. 封面与图文素材：封面图、菜谱图集、视频链接 / 视频上传、标签。
3. 食材配料：食材名称、用量、单位、备注、新增食材行、删除食材行、批量导入食材。
4. 调料配比：调料名称、用量、单位、新增调料行、删除调料行。
5. 制作步骤：步骤序号、步骤说明、步骤配图、新增步骤、删除步骤、拖拽调整顺序。
6. 关联信息：关联食材、适用场景、关键词、来源 / 作者、审核备注。
7. 右侧 C 端预览：展示菜谱封面、名称、标签、热量、食材、步骤、底部按钮。

### 5.2 食材管理

参考：`docs/prototypes/01-新增食材页.png`

新增食材页 / 编辑食材页需要包含：

1. 基础信息：食材名称、分类、食材类型、别名、产地、季节、关键词、简介。
2. 食材类型包括：蔬菜 / 肉类 / 水产 / 豆制品 / 谷物 / 其他。
3. 封面与图片：封面图、图片集、视频链接 / 视频上传。
4. 营养成分：热量、蛋白质、脂肪、碳水化合物、膳食纤维、钠、维生素、矿物质、添加自定义营养项。
5. 储存与处理：储存方式、保存期限、处理建议、不宜人群。
6. 其他信息：适用场景、推荐搭配食材、排序、状态。
7. 右侧 C 端预览：展示食材封面、名称、标签、营养摘要、适合人群、储存建议、常见做法。

### 5.3 水果管理

参考：`docs/prototypes/04-新增水果页.png`

新增水果页 / 编辑水果页需要包含：

1. 基础信息：水果名称、水果副标题、水果分类、口感、上市季节、适宜人群、规格 / 单位、参考热量、发布状态、是否推荐、水果简介。
2. 封面与图文素材：封面图、水果图集、视频链接 / 视频上传、标签。
3. 营养与属性：主要营养成分 / 特点、热量、糖分、维生素 C、膳食纤维、蛋白质、脂肪、产地、储存方式、保鲜期 / 建议食用期、食用方式、适合搭配、忌口提示。
4. 挑选方式 / 食用说明：说明卡片、可添加多条、支持排序、支持删除。
5. 关联信息：关联菜谱、适用场景、关键词、来源 / 作者。
6. 右侧 C 端预览：展示水果封面、名称、标签、热量、营养亮点、挑选建议、储存方式、适合搭配、相关菜谱。

### 5.4 调料管理

参考：`docs/prototypes/03-新增调料页.png`

新增调料页 / 编辑调料页需要包含：

1. 基础信息：调料名称、调料副标题、调料分类、口味、适用菜系、规格、品牌、发布状态、是否推荐、调料简介 / 描述。
2. 封面与图文素材：封面图、调料图集、视频链接 / 视频上传、标签。
3. 规格与属性：主要成分 / 配料、储存方式、保质期、产地、营养信息、适用搭配、适用人群 / 忌口提示。
4. 使用说明 / 内容详情：使用说明步骤、支持新增说明、支持删除、支持排序、支持配图。
5. 关联信息：关联菜谱、适用场景、关键词、来源 / 作者、审核备注。
6. 右侧 C 端预览：展示调料封面、名称、标签、简介、规格、品牌、储存方式、保质期、产地、适用搭配、相关菜谱。

### 5.5 酒水管理

参考：`docs/prototypes/02-新增酒水页.png`

新增酒水页 / 编辑酒水页需要包含：

1. 基础信息：酒水名称、酒水副标题、酒水分类、酒水类型、产地、品牌、容量规格、酒精度、甜度、口感、发布状态、是否推荐、酒水简介 / 描述。

5. 规格与属性：建议饮用温度、包装形式、保质期、原料 / 配料、营养信息、推荐搭配。
6. 饮用说明 / 内容详情：饮用说明、支持多条、支持配图、支持删除、支持排序。
7. 关联信息：关联分类、适用场景、关键词、来源 / 作者、审核备注。

## 8. 后台到 C 端的全链路要求

需要跑通以下流程：

```text
后台新增内容
→ 后台保存到数据库
→ 后台列表展示
→ C 端接口读取
→ C 端页面展示
→ 后台修改状态
→ C 端展示同步变化
```

至少需要验证：

1. 后台新增食材“番茄”，C 端食材列表能看到。
2. 后台编辑食材“番茄”，C 端详情同步变化。
3. 后台下架食材“番茄”，C 端不再展示。
4. 后台新增菜谱“番茄炒蛋”，C 端菜谱列表能看到。
5. 后台编辑菜谱“番茄炒蛋”，C 端详情同步变化。
6. 后台下架菜谱“番茄炒蛋”，C 端不再展示。
7. 后台新增水果，C 端水果列表能看到。
8. 后台新增调料，C 端调料列表能看到。
9. 后台新增酒水，C 端酒水列表能看到。
10. 后台新增分类，内容新增页可选择该分类。
11. 后台新增标签，内容新增页可选择该标签。
12. 后台新增单位，菜谱配料 / 调料配比里可选择该单位。

5. 修复后重新运行测试。
6. 重复直到核心链路通过。
7. 最后输出仍未通过的问题和原因。
