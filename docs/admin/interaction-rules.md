# 后台交互规则

## 统一交互

- 列表页必须支持搜索、筛选、分页。
- 新增、编辑、删除、上下架、排序必须有真实交互。
- 删除必须二次确认。
- 表单提交必须有 loading、防重复点击、成功和失败反馈。
- 接口失败必须展示明确错误信息。

## 列表页

- 表格使用 Ant Design Table 或 ProComponents。
- 操作列保持固定宽度。
- 关键状态使用 Tag 展示。
- 长文本使用省略和详情入口，不直接撑破表格。

## 表单页

- 表单使用 Ant Design Form。
- 必填项必须配置校验。
- 图片、视频、文件上传使用统一上传组件。
- 保存后返回列表或进入详情，行为必须一致。

## Drawer / Modal

- 轻量详情、配置项可使用 Drawer。
- 删除确认、批量操作确认使用 Modal。
- 不在 Modal 中承载过长复杂表单。

## 跳转规则

- 列表 → 详情 → 编辑路径必须清晰。
- 新增成功后跳转到详情或列表，并给出成功提示。
- 后台配置影响 C 端时，必须说明 C 端验证入口。
