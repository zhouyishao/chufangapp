# 后台配置 → C 端展示 验证清单

每次后台新增或修改以下内容后，必须在 C 端验证同步展示。

## 验证方法

1. 启动 server：`cd server && npm run dev`
2. 启动后台：`cd admin-frontend && npm run dev`
3. 启动 C 端：`cd frontend && npm run dev:h5`
4. 在后台操作后，刷新 C 端页面确认数据展示

## 检查清单

### 顶部导航
- [ ] 后台新增/编辑/启用/停用导航后，C 端首页顶部 Tab 是否同步
- [ ] 默认导航的内容是否在 C 端正确展示
- [ ] 停用导航后 C 端不再显示

### 首页模块
- [ ] 后台配置的首页模块内容是否在 C 端正确展示
- [ ] 模块排序是否正确

### Banner
- [ ] 后台新增/编辑 Banner 后，C 端首页轮播图是否同步
- [ ] Banner 图片是否正常加载
- [ ] 点击 Banner 是否跳转正确

### 推荐位
- [ ] 推荐菜谱是否在 C 端"今日推荐"展示
- [ ] 推荐食材是否在 C 端"时令食材"展示

### 菜谱管理
- [ ] 新增菜谱 → 提交审核 → 审核通过 → 上架 → C 端菜谱列表可见
- [ ] 菜谱封面图正常显示
- [ ] 菜谱详情：标题、食材、步骤、封面正确
- [ ] 下架后 C 端不可见

### 食材管理
- [ ] 新增食材 → 发布 → C 端食材列表可见
- [ ] 食材详情：封面、营养、挑选技巧正确
- [ ] 下架后 C 端不可见

### 水果管理
- [ ] 新增水果 → 发布 → C 端可见
- [ ] 封面图正常显示

### 调料管理
- [ ] 新增调料 → 发布 → C 端可见

### 酒水管理
- [ ] 新增酒水 → 发布 → C 端可见

### 分类管理
- [ ] 新增分类后，菜谱/食材表单下拉可选
- [ ] 禁用分类后，表单下拉不再显示
- [ ] C 端按分类筛选正常

### 标签管理
- [ ] 新增标签后，菜谱/食材表单可选
- [ ] C 端标签展示正确

### 单位管理
- [ ] 新增单位后可用
- [ ] 换算关系正确

### 菜篮子
- [ ] 添加到菜篮子功能正常
- [ ] 采购清单展示正确

## 通用检查

- [ ] 所有图片正常加载（不是裂图）
- [ ] 所有空值有合理展示（不是 null 或 undefined）
- [ ] 列表分页正常
- [ ] 接口错误时有重试按钮
- [ ] C 端没有使用假数据
