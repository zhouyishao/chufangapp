# C 端 UI 设计规范

## 1. 设备基准

- 基准设备：iPhone 15（393 × 852 pt）
- 不要手机壳效果
- 不需要模拟顶部状态栏硬件边框

## 2. 设计风格

高级感、极简、奶油侘寂风（Cream Wabi-Sabi）。

关键词：安静、克制、成熟、日常、真实、不炫技。

## 3. 配色系统

| 用途 | 色值 | 说明 |
|------|------|------|
| 主色 | `#7A8B6F` | 鼠尾草绿，按钮、选中态、强调 |
| 背景 | `#F5F1EA` | 暖奶油色，页面底色 |
| 卡片 | `#FFFDFC` | 暖白，卡片和列表项 |
| 主文字 | `#2F2F2F` | 深灰，标题和正文 |
| 辅助灰 | `#B7AEA1` | 次要文字、分割线 |
| 点缀橙 | `#C27B48` | 极少使用，仅用于价格、重要标签、收藏 |

## 4. 排版

- 页面不能拥挤
- 卡片不能溢出屏幕
- 标签不能超出卡片边界
- 标题不超过 2 行
- 描述文字不超过 3 行
- 列表间距统一、呼吸感强

## 5. 组件

- 卡片圆角 12-16px
- 阴影极轻，优先用边框代替重阴影
- 按钮圆角 24-28px
- 输入框圆角 12px
- 图片比例统一 4:3 或 1:1
- 底部导航 icon 简洁有特点，不能花哨

## 6. 图片

- 食材图、菜谱图优先级高于大段文字
- 封面图比例统一
- 加载失败显示灰色占位图
- 空图显示"暂无图片"
- 所有图片地址通过 `resolveAssetUrl` 统一处理

## 7. 空状态 / 错误状态

所有页面必须包含：
1. Loading 状态
2. Empty 空状态（带说明文字）
3. Error 错误状态（带重试按钮）
4. 不允许接口失败后白屏

## 8. 禁止

- 禁止使用纯黑 `#000000`
- 禁止使用高饱和颜色
- 禁止使用霓虹色、荧光色
- 禁止重阴影、多层阴影
- 禁止复杂渐变
- 禁止花哨动画
- 禁止 emoji 作为主要图标
- 禁止页面像低保真线框图
- 禁止页面像未完成的 Demo
