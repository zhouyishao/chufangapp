# Codex 任务模板

## 任务类型

- C 端 UI / 后台 / 后端 / 数据库 / 文档 / 联调。

## 任务目标

用 1-3 句话说明本次要完成什么，不写无关背景。

## 必读文档

- `AGENTS.md`
- `docs/codex/coding-rules.md`
- 按任务类型补充对应文档和 skill。

## 修改范围

列出允许修改的目录和文件，未列出的目录默认不改。

## 验收标准

- 页面是否打开。
- 交互是否闭环。
- 接口是否返回正确。
- 数据是否落库。
- 构建或类型检查是否通过。

## 输出格式

1. 本次使用了哪些 skill。
2. 修改了哪些文件。
3. 每个文件改了什么。
4. 如何启动测试。
5. 是否影响接口 / 数据库 / 页面。
6. 风险点。
