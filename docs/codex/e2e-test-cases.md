# E2E 测试用例

## 后台基础交互

- 登录成功后进入后台。
- 菜单可跳转。
- 列表搜索、筛选、分页可用。
- 新增、编辑、删除有真实反馈。

## 内容管理

- 菜谱：新增 → 编辑 → 上架 → C 端可见 → 下架 → C 端不可见。
- 食材：新增 → 发布 → C 端列表和详情可见。
- 水果、调料、酒水：新增后对应列表可见。

## 分类标签单位

- 新增分类后，内容表单可选择。
- 禁用分类后，内容表单不再显示。
- 标签新增后，C 端展示正确。
- 单位新增后，价格和用量可使用。

## 首页运营

- 后台配置顶部导航后，C 端首页同步。
- Banner 图片正常加载，点击跳转正确。
- 推荐位排序和上下架生效。

## 资源管理

- 图片上传后后台可预览。
- C 端使用上传资源正常显示。
- API 接口配置保存后列表可见。

## C 端展示

- 首页加载无白屏。
- 菜谱列表和详情字段完整。
- 食材列表和详情字段完整。
- 收藏、浏览、菜篮子基础交互可用。

## 失败处理

- 测试失败必须记录失败页面、操作步骤、错误信息、修复结果。
