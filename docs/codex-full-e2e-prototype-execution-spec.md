## 1. 本轮核心目标

本轮不是继续只画静态页面，也不是只跑 `npm run build`。

本轮目标是完成：

```text
后台页面功能可用
  ↓
后台接口可用
  ↓
后台数据写入数据库
  ↓
后台列表读取真实数据
  ↓
C 端接口读取后台已发布数据
  ↓
C 端页面真实展示
  ↓
Codex 使用真实页面自动化点击测试
  ↓
发现问题后修复
  ↓
重复测试直到核心链路通过
```

必须重点修复当前问题：

1. 菜谱、食材、水果、调料、酒水仍然是假数据。
2. 后台保存的数据没有真正展示到 C 端。
3. C 端页面目前没有正常展示后台或假数据。
4. 资源接入中心还没有真正接到后台资源库。
5. API 接口管理页面需要新增「接口配置列表页」和「新增接口页」。
6. 面包屑、搜索、查询、重置、分页、新增、编辑、查看、删除、启用、禁用、保存草稿、保存并发布等很多按钮需要逐页测试并修复。
7. Codex 需要在真实页面里自动点击测试，测试失败后直接修复并继续测试。

## 3. 重要执行规则

### 3.1 开始前必须检查

开始前先执行：

```bash
git status
```

并检查：

1. 当前工作区是否有未提交改动。
2. 前端项目目录在哪里。
3. 后端项目目录在哪里。
4. C 端项目目录在哪里。
5. 数据库是否能连接。
6. 是否已有 ORM。
7. 是否已有接口文档。
8. 是否已有 Playwright / Cypress。
9. 当前页面哪些地方仍然是假数据。

### 3.2 不要做的事

1. 不要删除已有页面。
2. 不要重构整个项目。
3. 不要大改侧边栏、顶部栏、公共布局。
4. 不要把新增 / 编辑页改成弹窗。
5. 不要为了图省事继续保留前端假数据。
6. 不要只跑 build 就说完成。
7. 不要只修改后台，不检查 C 端展示。
8. 不要频繁询问普通实现细节，能自动继续就自动继续。
9. 不要清空数据库。
10. 不要删除用户现有代码和文件。

### 3.3 允许自动继续的情况

以下问题 Codex 应直接修复，不要中断询问：

1. 路由不存在。
2. 按钮未绑定事件。
3. 页面无数据。
4. 前端字段和接口字段不一致。
5. 保存后列表不刷新。
6. 面包屑不可点击。
7. 查询、重置、分页不可用。
8. C 端没有调用接口。
9. 后端接口不存在。
10. 数据库字段缺失。
11. Playwright 测试失败。
12. `npm run build` 报错。

## 14. 修复循环要求

Codex 必须按以下循环执行：

```text
启动项目
  ↓
打开真实页面
  ↓
执行自动化测试
  ↓
发现失败
  ↓
查看日志/截图/报错
  ↓
定位原因
  ↓
修复代码
  ↓
重新测试
  ↓
直到核心链路通过
```

不能出现：

1. 发现测试失败后直接停止。
2. 只报告问题不修复。
3. 只运行 build 不跑页面。
4. 只改后台不测试 C 端。
5. 只改 UI 不接接口。
6. 只接假数据不接数据库。
7. 只回复“完成”没有测试报告。
