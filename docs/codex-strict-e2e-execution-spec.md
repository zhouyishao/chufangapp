## 5. 页面统一结构要求

所有新增 / 编辑页统一结构：

```text
左侧后台侧边栏
顶部面包屑 + 搜索框 + 用户信息
页面标题 + 页面说明
右上角操作按钮：取消 / 保存草稿 / 保存并发布
主内容区：分块卡片表单
右侧：C 端内容预览
底部或顶部固定操作区
```

按钮要求：

| 按钮 | 行为 |
|---|---|
| 取消 | 返回对应列表页；未保存内容不生效 |
| 保存草稿 | 保存为 draft，不在 C 端展示 |
| 保存并发布 | 保存为 published / enabled，可在 C 端展示 |
| 编辑 | 进入编辑页并回显数据 |
| 删除 | 二次确认后软删除 |
| 启用 / 禁用 | 真实改变状态 |
| 上架 / 下架 | 真实改变发布状态 |
| 查询 | 使用筛选条件查询真实数据 |
| 重置 | 清空筛选条件并刷新列表 |
| 分页 | 真实分页，不只是 UI 变化 |

### 9.2 Playwright 基础要求

需要支持：

1. 自动启动或连接后台前端。
2. 自动启动或连接 C 端前端。
3. 自动启动或连接后端服务。
4. 失败时保留 screenshot / video / trace。
5. 测试数据使用唯一前缀，避免重复冲突。

测试数据前缀建议：

```text
E2E_时间戳_
```

例如：

```text
E2E_20260603_番茄
E2E_20260603_番茄炒蛋
```
