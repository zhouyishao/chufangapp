# Figma Prototype to Mobile UI Skill

## 什么时候使用本 Skill

当任务涉及以下内容时，必须使用本 Skill：

- 根据用户提供的 Figma 原型图修改 C 端页面
- 根据用户提供的截图原型图实现页面
- 在 Figma 中绘制高保真 C 端原型图
- 将 Figma 页面还原成前端代码
- 优化 C 端首页、食材详情页、菜谱详情页、菜篮子、搜索页、我的页面
- 检查页面是否和原型图一致
- 把现有页面改成和原型图一致
- 处理 Figma MCP、Figma node、设计 token、图片素材、组件还原相关任务

## 最高优先级原则

用户已经有明确原型图。

本 Skill 的最高优先级是：**按用户原型图还原，不自由发挥。**

必须遵守：

1. 不要重新设计。
2. 不要擅自改变页面结构。
3. 不要新增原型图里没有的复杂模块。
4. 不要把“优化 UI”理解成自由创作。
5. 如果当前代码和原型图不一致，以原型图为准。
6. 如果旧设计和新原型图不一致，以新原型图为准。
7. 页面布局、模块顺序、间距、圆角、字体层级、颜色、卡片样式、按钮样式、底部导航样式，都必须尽量按原型图还原。
8. 页面数据必须继续使用真实接口，不允许回退成 mock、fake、static 假数据。
9. 所有关键 UI 元素必须是可维护的组件或样式，不要用整张截图当页面背景。
10. 完成后必须输出差异清单和接口绑定清单。

## 设计系统

### 颜色

优先使用用户原型图中的颜色。  
如果原型图没有明确颜色，使用以下项目色：

- 页面背景：#F8F4EC
- 米白背景：#F7F3EA
- 主色绿色：#6F8F63
- 深绿色：#4F7048
- 浅绿色标签：#EAF4E5
- 卡片白色：#FFFFFF
- 正文：#2F2F2F
- 次级文字：#777777
- 弱提示文字：#A0A0A0
- 边框：#E8E0D3
- 危险色：#E53935

### 字体

移动端字体层级：

- 页面主标题：24px / 600
- 模块标题：20px / 600
- 卡片标题：15px-16px / 600
- 正文：14px-15px / 400
- 辅助信息：12px-13px / 400
- 标签文字：11px-12px / 500
- 底部导航文字：11px-12px / 500

如果原型图中字体层级不同，以原型图为准。

### 圆角

- 页面大卡片：24px
- 内容卡片：16px
- 图片卡片：14px
- 搜索框：999px
- 标签：999px
- 主按钮：999px 或 12px
- 底部导航容器：28px

### 间距

移动端默认：

- 页面左右边距：16px
- 模块上下间距：20px
- 卡片内边距：12px-16px
- 标题与内容间距：12px
- 图片与文字间距：8px
- 卡片之间间距：12px
- 底部安全区：34px

如果原型图中的间距更明确，以原型图为准。

## Figma / 原型图还原流程

每次开始实现前，必须先做分析，不要直接写代码。

### 第一步：识别原型图

先判断用户提供的是：

- Figma 链接
- Figma node
- 截图原型图
- 已导出的 PNG/JPG
- 旧页面截图
- 新页面目标图

然后输出：

- 页面名称
- 页面状态
- 页面模块
- 是否已有对应代码页面
- 当前页面与原型图差异

### 第二步：拆解页面结构

必须拆解：

- 顶部状态栏
- 顶部搜索
- 顶部 Tab
- Hero 区域
- 内容卡片
- 横向滚动卡片
- 底部导航
- 固定按钮
- 空状态 / 加载状态 / 错误状态

### 第三步：抽取组件

优先复用或创建以下组件：

- AppHeader
- SearchBar
- TopTabs
- HeroBanner
- SectionTitle
- RecipeCard
- IngredientCard
- CategoryPill
- Tag
- BottomNav
- NutritionGrid
- SelectionGuide
- RecipeStepList
- BasketButton
- EmptyState
- LoadingState
- ErrorState

禁止每个页面重复写一套样式。

### 第四步：按原型图实现

实现时按以下优先级：

1. 结构还原
2. 尺寸还原
3. 间距还原
4. 字体层级还原
5. 颜色还原
6. 圆角和阴影还原
7. 图片比例还原
8. 动效和交互状态还原
9. 接口绑定
10. 数据异常状态处理

## C 端首页规范

首页原型包含两个常见状态：

### 初始状态

特点：

- 顶部沉浸式大图
- 搜索框叠加在大图上
- 右侧加号按钮
- 顶部 Tab 叠加在大图区域
- 当前 Tab 使用绿色文字和下划线
- Hero 区展示菜名、副标题、按钮、轮播点
- 下方内容区域为白色上浮圆角卡片
- 展示「为你推荐」
- 展示「时令食材」
- 底部导航为白色悬浮胶囊

### 滑动后状态

特点：

- 搜索与导航吸顶
- 顶部背景从大图过渡为模糊色块或浅色背景
- 内容区上移
- 展示更多模块
- 底部导航固定

首页模块建议：

- 推荐
- 今日吃什么
- 时令食材
- 家庭菜谱
- 为你推荐
- 时令食材
- 热门分类
- 厨房小技巧
- 热门合集

如果原型图只展示部分模块，只实现原型图中出现的模块。

## 食材详情页规范

食材详情页必须支持后台的 selectionGuide 字段。

页面结构：

1. 顶部食材大图
2. 食材名称
3. 标签
4. 简介
5. 营养成分
6. 怎么挑
7. 储存建议
8. 常见做法
9. 相关菜谱
10. 加入菜篮子

selectionGuide 数据结构：

```json
[
  {
    "groupTitle": "外观",
    "sort": 1,
    "items": [
      {
        "title": "看大小形状",
        "content": "一般而言，个头越大、外形越匀称的越好。",
        "imageUrl": "",
        "sort": 1
      }
    ]
  }
]
```

展示规则：

1. selectionGuide 为空时，不展示「怎么挑」模块。
2. 有内容时，展示「怎么挑」标题。
3. 先展示 groupTitle，例如：外观、气味、手感。
4. 每个 groupTitle 下展示多个 item。
5. 每个 item 展示 title、content、imageUrl。
6. 没有图片时只展示文字。
7. 空分组不展示。
8. 空 item 不展示。
9. 按 sort 升序展示。
10. 不允许写死外观、手感、气味，必须读取接口返回。

## 菜谱详情页规范

页面结构：

1. 顶部菜谱大图
2. 菜谱名称
3. 标签
4. 烹饪信息：时间、难度、人数
5. 食材清单
6. 制作步骤
7. 小贴士
8. 相关推荐
9. 加入菜篮子

制作步骤展示：

- 使用步骤编号
- 每一步支持文字
- 每一步支持配图
- 图片圆角统一
- 步骤间距清晰
- 不要大段文字堆叠

## 接口要求

所有 C 端页面必须读取真实接口。

必须检查：

- GET /api/app/home
- GET /api/app/recipes
- GET /api/app/recipes/:id
- GET /api/app/ingredients
- GET /api/app/ingredients/:id
- GET /api/app/categories
- GET /api/app/search
- GET /api/app/basket

禁止使用：

- mock
- mockData
- fakeData
- demoData
- staticData
- sampleData
- 写死数组冒充接口数据

如果发现假数据：

1. 输出假数据位置。
2. 替换为真实接口请求。
3. 补齐 loading、empty、error 状态。
4. 验证刷新页面后数据不丢失。

## 交互要求

首页：

- 搜索框点击进入搜索页
- 顶部 Tab 可切换
- Hero 按钮进入菜谱详情或专题页
- 推荐卡片点击进入菜谱详情
- 食材卡片点击进入食材详情
- 更多按钮进入对应列表页
- 底部导航可切换页面

食材详情：

- 点击加入菜篮子
- 点击相关菜谱进入菜谱详情
- 怎么挑内容按接口展示
- 图片可以正常加载
- 空字段不展示

菜谱详情：

- 点击加入菜篮子
- 食材清单可加入菜篮子
- 推荐菜谱可跳转
- 步骤图正常展示

## 输出要求

完成任务后必须输出：

1. 已处理的页面清单。
2. 对应原型图来源。
3. 已创建或复用的组件。
4. 当前页面与原型图仍存在的差异。
5. 差异原因。
6. 每个页面绑定的接口。
7. 已删除或替换的假数据位置。
8. 是否通过 lint。
9. 是否通过 build。
10. 是否达到「按原型图还原 + 真实接口展示」标准。

## 自查清单

提交前必须自查：

- 页面是否和原型图结构一致
- 模块顺序是否一致
- 顶部搜索是否一致
- Tab 是否一致
- 卡片圆角是否一致
- 图片比例是否一致
- 底部导航是否一致
- 字体层级是否一致
- 页面间距是否一致
- 是否复用公共组件
- 是否接真实接口
- 是否没有 mock/fake/static 数据
- 是否有 loading 状态
- 是否有 empty 状态
- 是否有 error 状态
- 是否无控制台报错
- 是否移动端适配正常
- 是否 lint/build 通过

## 禁止事项

禁止：

- 不看原型图直接改
- 自由发挥重新设计
- 把页面做成另一种风格
- 擅自新增模块
- 擅自删除原型图已有模块
- 用整张截图作为页面背景
- 每个页面写一套独立样式
- 写死假数据
- 忽略接口字段
- 忽略移动端安全区
- 忽略底部导航固定
- 忽略空状态
- 忽略加载状态
- 忽略错误状态
